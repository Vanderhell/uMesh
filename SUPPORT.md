# Support

Use this repository's issue templates for actionable reports.

## For questions

Use GitHub Discussions for usage questions, design questions, and general help.

## For bugs and regressions

Open a bug report and include:

- commit SHA
- platform
- compiler or toolchain
- CMake options
- exact command
- expected behavior
- actual behavior
- logs
- whether the issue is host-simulated, ESP32 compile, or real hardware

## For security issues

Use the security report template and keep details minimal until triage.

## For verification failures

If a documented gate fails, use the verification-failure template and attach the exact command and output.
